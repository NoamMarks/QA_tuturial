1. Relational Databases

דוגמה: MySQL, PostgreSQL, SQL Server
שימוש: נתונים מובנים עם קשרים חזקים
יתרון: עקביות, תמיכה מלאה ב-ACID ושאילתות SQL מורכבות
חיסרון: פחות גמיש לשינויים במבנה, קשה יותר לסקייל

2. In-Memory Databases

דוגמה: Redis, Memcached
שימוש: מהירות תגובה קיצונית, מטמון, סשנים
יתרון: מהיר במיוחד, תמיכה בהתמדה (Redis)
חיסרון: יקר בזיכרון, מוגבל בנפח הנתונים

3. Key-Value Stores

דוגמה: DynamoDB, Redis (כ-KV store)
שימוש: גישה פשוטה לפי מפתח ייחודי
יתרון: סופר מהיר, סקיילביליטי מצוינת
חיסרון: לא תומך בשאילתות מורכבות או יחסים

4. Document Databases

דוגמה: MongoDB, CouchDB
שימוש: גמישות במבנה הנתונים, אפליקציות JSON-based
יתרון: תמיכה מובנית ב-JSON, שאילתות על תוכן המסמך
חיסרון: פחות יעיל ליחסים מורכבים, עלולים לגרום לכפילות נתונים

5. Wide-Column Stores

דוגמה: Cassandra, HBase
שימוש: כתיבה וקריאה מהירה בסקייל גבוה
יתרון: סקיילבילי מאוד, ביצועים גבוהים
חיסרון: מורכבות בעיצוב הסכימה ובניהול

6. Graph Databases

דוגמה: Neo4j, Amazon Neptune
שימוש: ניתוח קשרים מורכבים בין ישויות
יתרון: אידיאלי לרשתות חברתיות, המלצות, גילוי הונאות
חיסרון: לא מתאים לנתונים שטוחים, ביצועים איטיים יותר בשאילתות פשוטות

7. Time-Series Databases

דוגמה: InfluxDB, TimescaleDB
שימוש: נתוני ניטור, מדדים, IoT
יתרון: מותאם לנתונים תלויי זמן, דחיסה יעילה
חיסרון: מקרה שימוש ייחודי, לא מתאים לשאילתות כלליות

8. Search Databases

דוגמה: Elasticsearch, Solr
שימוש: חיפושי טקסט חופשי, ניתוח לוגים
יתרון: חיפוש חכם ומהיר, תמיכה בטקסט לא מובנה
חיסרון: לא בסיס נתונים ראשי, צורך בסינכרון עם DB אחר

9. Vector Databases

דוגמה: Pinecone, Weaviate, FAISS
שימוש: AI, חיפוש דמיון, המלצות מבוססות ML
יתרון: מותאם לוקטורים, תמיכה בחיפוש קרבה
חיסרון: מתמחה רק בנתונים וקטוריים, עלות גבוהה

10. Distributed Databases

דוגמה: CockroachDB, Spanner
שימוש: זמינות ועמידות גבוהה, אפליקציות גלובליות
יתרון: מבוזר, עמיד לכשלים, סקיילביליטי גבוהה
חיסרון: תפעול מורכב, עלות גבוהה, latency גבוהה יותר

שאלון לתרגול

  • באיזה סוג DB תשתמש למערכת לוגים בזמן אמת?
    תשובה: Time-Series DB כמו InfluxDB או Search DB כמו Elasticsearch
  • מה ההבדל המרכזי בין Document ל-Key-Value?
    תשובה: Document DB מבין את מבנה הנתונים (JSON) ומאפשר שאילתות על התוכן, בעוד Key-Value רק מחזיר ערך שלם לפי מפתח
  • למה Graph DB עדיף על Relational במערכת קשרים חברתיים?
    תשובה: Graph DB מותאם לחיפוש קשרים רב-שכבתיים (חברים של חברים) ביעילות רבה יותר מ-JOINs מורכבים
